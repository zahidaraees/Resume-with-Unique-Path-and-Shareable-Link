# Contributing to Resume Builder Pro

Thank you for your interest in contributing! 🎉 We welcome all contributions, whether it's a bug fix, new feature, or documentation improvement.

## 🚀 Quick Start

1. **Fork** the repository
2. **Clone** your fork locally
3. **Create** a new branch for your feature
4. **Make** your changes
5. **Test** your changes locally
6. **Submit** a pull request

## 🐛 Reporting Bugs

Before creating bug reports, please check existing issues to avoid duplicates.

### When submitting a bug report, include:

- **Clear title and description**
- **Steps to reproduce** the behavior
- **Expected vs actual** behavior
- **Screenshots** if applicable
- **Browser and OS** information
- **Console errors** if any

### Example:

```markdown
**Describe the bug**
Resume doesn't generate when clicking the button on Safari.

**To Reproduce**
1. Go to '...'
2. Click on '....'
3. Scroll down to '....'
4. See error

**Expected behavior**
Resume should generate and display immediately.

**Screenshots**
If applicable, add screenshots here.

**Environment:**
- OS: macOS 14.0
- Browser: Safari 17.0
```

## 💡 Suggesting Features

We love hearing new ideas! Before suggesting a feature:

- Check if it's already been proposed
- Explain the **problem** it solves
- Describe the **benefit** to users
- Keep it **focused and specific**

## 🎨 Code Style Guidelines

### HTML
- Use semantic HTML5 elements
- Maintain proper indentation (2 spaces)
- Add comments for major sections

### CSS
- Use CSS variables for colors
- Follow BEM naming convention where possible
- Keep selectors specific but not overly complex
- Add comments for sections

### JavaScript
- Use `const` and `let` (avoid `var`)
- Use camelCase for variables and functions
- Add JSDoc comments for functions
- Handle errors gracefully

## 📝 Pull Request Process

1. Update the README.md with details of changes if needed
2. Test your changes in multiple browsers
3. Ensure your code follows the style guidelines
4. Update the CHANGELOG.md if applicable
5. Your PR will be reviewed before merging
